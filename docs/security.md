# 🔒 Безопасность

Обзор мер базовой безопасности, настроенных в приложении, и того, где они находятся в коде.

## 🌐 CORS

Разрешённые источники, методы и заголовки задаются через `.env` (`CORS_ORIGINS`, `CORS_METHODS`, `CORS_HEADERS`) вместо `"*"`, чтобы браузер не мог слать запросы к API с произвольных доменов произвольными методами.

Настроено в [app/main.py](../app/main.py).

## 🛡️ HTTP security-заголовки

На каждый ответ добавляются заголовки:

- `X-Content-Type-Options: nosniff` — запрещает браузеру угадывать MIME-тип содержимого.
- `X-Frame-Options: DENY` — запрещает встраивание страниц API во `<iframe>` (защита от clickjacking).
- `Referrer-Policy: strict-origin-when-cross-origin` — ограничивает, что попадает в заголовок Referer при переходе на другие сайты.
- `Permissions-Policy` — отключает доступ браузера к геолокации, микрофону и камере.
- `Strict-Transport-Security` — заставляет браузер ходить только по HTTPS (эффективно, когда приложение реально работает под HTTPS).

Реализовано в [app/core/security_headers.py](../app/core/security_headers.py), подключено как middleware в [app/main.py](../app/main.py).

## 🙈 Скрытие Swagger/OpenAPI в продакшене

Когда `DEBUG=false`, эндпоинты `/docs`, `/redoc` и `/openapi.json` отключены (возвращают 404) — схема API не должна быть публично доступна в продакшене.

Для локальной разработки поставьте `DEBUG=true` в `.env`.

Настроено в [app/main.py](../app/main.py) через `docs_url`/`redoc_url`/`openapi_url`.

## 🚧 TrustedHostMiddleware

Сервер принимает запросы только с Host-заголовком из списка `ALLOWED_HOSTS` (`.env`). Запросы с другим Host отклоняются с 400 — защита от Host header атак (например, отравление кэша или подмена ссылок в письмах, сгенерированных по значению Host).

`testserver` включён в список по умолчанию, чтобы не ломать `TestClient` в тестах — реальный Host у него всегда фиксированный.

## ⏱️ Rate limiting

Ограничение частоты запросов с одного IP через [slowapi](https://github.com/laurentS/slowapi), лимит задаётся `RATE_LIMIT` в `.env` (по умолчанию `100/minute`). При превышении — `429 Too Many Requests`.

Реализовано в [app/core/limiter.py](../app/core/limiter.py).

**Важно:** лимит навешивается декоратором `@limiter.limit(...)` на каждый роут отдельно (см. [app/routers/http/root.py](../app/routers/http/root.py)), а не глобально через middleware. В используемой версии FastAPI роутер собирается лениво, из-за чего `app.routes` не обходится штатным образом — автоматическое определение эндпоинтов через `SlowAPIMiddleware` не срабатывает. Поэтому при добавлении нового роутера не забудьте явно повесить декоратор `@limiter.limit(settings.rate_limit)` на новые эндпоинты.

## Секреты и `.env`

`.env` содержит реальные значения окружения и исключён из git через `.gitignore`. В репозиторий попадает только `.env.example` с примерами значений (без секретов). При добавлении новых чувствительных настроек (пароли, ключи API и т.п.) — то же самое: значение в `.env`, пример без секрета — в `.env.example`.

## 🚫 Что не входит в этот базовый набор

Ниже — то, что стоит добавить отдельно при появлении соответствующей функциональности, но что не настроено сейчас, так как в приложении пока нет пользователей и большинства внешних интеграций (кроме подключения к PostgreSQL, см. [docs/database.md](database.md)):

- Аутентификация и авторизация (JWT/OAuth2/сессии).
- Валидация и хранение секретов через внешний secret-manager (Vault, AWS Secrets Manager и т.п.) вместо `.env`.
- HTTPS/TLS — предполагается, что терминируется на уровне reverse-proxy (nginx, ALB и т.п.), не в самом приложении.
- Логирование и мониторинг попыток атак (WAF, IDS).
